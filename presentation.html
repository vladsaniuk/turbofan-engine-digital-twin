What we built

8 Production-Grade Features

Every feature is independently functional. The telemetry pipeline is feature-gated — the app runs identically without it.

🤖

ML-based RUL Prediction

A Random Forest model trained on NASA C-MAPSS FD001 run-to-failure data. Predicts Remaining Useful Life per cycle. Trained on 100 engine units with RUL capped at 130 cycles. Model persists to disk and auto-retrains on first run.

twin/model.py
scikit-learn
💬

LLM Engine Advisor

Gemini-powered chat interface embedded in the dashboard. Ask natural language questions about sensor trends, failure risk, and what-if scenarios. Context includes current state, 10-cycle history, and subsystem health scores.

twin/llm.py
Gemini Flash
🕸️

Subsystem Health Graph

NetworkX directed graph models engine topology: Fleet → Engine → Subsystem → Sensor. Computes per-subsystem degradation score as mean percent drift over the last 20 cycles. Ranks Fan, HPC, LPC, LPT, HPT, Combustor.

twin/graph.py
NetworkX
📋

DTDL Contract Validation

Every tick is validated against an Azure Digital Twins Definition Language schema. Checks all Telemetry and Property fields are present and finite. Maps raw sensor IDs to DTDL names. Shows ✅ VALID or ❌ INVALID + violations in the UI.

twin/contract.py
DTDL Schema
🔧

Maintenance Scheduler

SimPy discrete-event simulation runs when predicted RUL drops below a configurable threshold. Simulates crew acquisition, queuing, and a 4-hour repair cycle. Returns urgency level (High / Critical) and a timestamped event log.

twin/maintenance.py
SimPy
📡

Streaming Telemetry Pipeline

Feature-gated RabbitMQ + InfluxDB pipeline. App publishes each tick via MQTT. Standalone consumer subscribes and writes to InfluxDB. Dashboard queries InfluxDB for real chart data. Zero impact on app when flag is OFF.

twin/mqtt_io.py
twin/consumer.py
twin/influx_io.py
✈️

Engine Visualisation

Two view modes: SVG schematic blocks with per-subsystem health colouring, and a realistic turbofan cutaway overlay rendered as interactive HTML. Subsystem colours shift from green to red as degradation increases.

twin/engine_svg.py
twin/engine_overlay.py
🎮

Interactive Playback

Play / Pause / Scrub controls over the full engine lifetime. Per-unit selection (1–100 engines). Configurable tick speed (0.1–2.0s). Sensor multiselect for custom chart views. Flat sensors auto-pruned at load time.

app.py
Streamlit

Run It

Quick Start

Basic mode runs with no infrastructure. Full pipeline takes one extra docker command.

Basic Mode
# 1. Create venv python -m venv .venv source .venv/bin/activate # 2. Install pip install -r requirements.txt # 3. Set API key export GEMINI_API_KEY="your-key" # 4. Run streamlit run app.py
Full Pipeline
# 1. Start RabbitMQ + InfluxDB docker compose up -d # 2. Start consumer (separate terminal) python -m twin.consumer # 3. Run app with pipeline export TWIN_ENABLE_PIPELINE=1 streamlit run app.py # 4. Hit Play from cycle 0 # → data flows into InfluxDB
